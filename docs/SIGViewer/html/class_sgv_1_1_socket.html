説明

ソケットの基本クラス

派生クラスにServerSocketとClientSocketがある．

ServerSocketは、自らが主体的にソケットをオープンする(サーバーとなる)場合に使う． ClientSocketは、サーバーに接続するためのソケットを作るときに使う．

int Sgv::Socket::recvPacket ( Sgv::Packet **  ppPacket  ) 

ソケットからデータを読み出し、パケットとして返す

引数:
ppPacket [i] ソケットから受信したデータ
戻り値:
ステータス値
 1 (=ソケットからデータを受信した)
 0 (=ソケットが閉じられた(ブロッキングモードの場合)、有効なデータが届いていない(=ノンブロッキングモードの場合))
 -1 (=エラー発生)

SIGVerseの通信データは、先頭が0xABCD（ヘッダマーク）で始まり、0xDCBA（テイルマーク）で終わる． このヘッダとテイルで囲まれた一連のデータブロックをパケットと呼ぶ．

このメソッドではヘッダとテイルの識別を行うため、パケットの受信が簡単に行える．

受け取ったパケットはppPacketに返る. 呼び出し側はppPacketをdeleteしなければならない

 Sgv::Packet *p = NULL;
 for (;;)
 {
    int status = recvPacket(&p);

    if (status < 0) {
            // エラー
            exit 1;
    } else if (status > 0) {
            // 受信完了．データはpに入っている．
            break;
        }
    }
 ...
 if (p) delete p;   // 受け取ったパケットを解放する

int Sgv::Socket::recvPacket_NonBlock ( Sgv::Packet **  ppPacket  )  [protected]

ソケットからデータを受け取り、パケットとして返す（ノンブロッキングモード）

引数:
ppPacket [o] 受信したデータ
戻り値:
ステータス値
 1 = 成功 (読み込んだデータはppPacketに返る）
 0 = データが届いていない
 -1 = エラー

ノンブロッキングモードの場合、ブロッキングモードと異なり、 ソケットからデータが届かなくてもリターンしてくる（その場合の返り値は0）

そのため、データが届かなかった場合もブロックせず、 他の処理を続けることができる．（以下のコード例を参照）

 Sgv::Packet *p = NULL;
 for (;;)
 {
    int status = recvPacket_Block(&p);

    if (status < 0) {
            // エラー
            exit 1;
    } else if (status > 0) {
            // 受信完了．データはpに入っている．
            break;
        } else {
            // データは届いていなかった．
            // 他の処理を行う．
        }
    }
 ...
 if (p) delete p;
